자바스크립트의 이벤트에 관하여

브라우저로 표시되는 페이지 상에서는 다양한 이벤트가 발생한다.

  • 버튼이 (더블)클릭 되었다.
  • 마우스 포인터가 문자열 위에 있다(벗어 났다)
  • 텍스트 박스의 내용이 변경

클라이언트 측 자바스크립트에서는 해당 이벤트에 따라 실행 되는 코드를 작성하는 것이 특징이고 이런 모델을 이벤트 드리븐 모델이라고 부른다. 이런 많은 이벤트들에 대응해 그 처리 내용을 정의하는 함수를 이벤트 핸들러라고 한다.

자바스크립트에는 많은 이벤트가 있으며 아래 참조 사이트에 정리가 잘 되어 있으니 참조 가능하능하다.
은혜로우신 MDN님

이벤트 리스너 정의 하기

결국 중요한 것은 세가지다.

  • 어느 요소에서 발생하는지
  • 어떤 이벤트를?
  • 어떻게 연관 시킬 것인지

자바스크립트에서는 이벤트를 정의하기 위해 몇 가지 방법을 제공하고 있다.

  1. HTML 태그내의 속성으로 선언하기
  2. 요소 객체의 속성으로 선언
  3. addEventListener 메서드 사용

현재는 거의 3번의 방법을 많이 사용한다. 게다가 동일한 요소/ 동일한 이벤트에 여러 이벤트 핸들러를 연결 시킬 수 없다는 단점이 있기도 하므로 여기서는 3번의 방법만 사용한다.

element.addEventListener(event_type, listener)

이벤트 핸들러를 이용해서 input 태그의 value 값을 가져왔다.

이벤트 버블링, 이벤트 위임

이벤트 버블링

이벤트 버블링이란 특정 화면 요소에 이벤트가 발생하면 해당 이벤트의 상위 화면 요소로 전달 되는 특성을 의미한다.

사진은 가장 아래 요소인 스트롱 태그에 이벤트가 전달되면 그 위의 요소인 td 태그로 이벤트가 전달되고 가장 상위 태그인 table 태그로 전달 되는 이벤트 버블링 현상을 보여 준다.

하나의 엘리먼트만 클릭 했을 뿐인데 이벤트 버블링으로 인하여 다른 엘리먼트들의 클래스들 값도 콘솔로그로 출력되는 걸 볼 수 있다.

이벤트 버블링은 브라우저가 이벤트를 감지하는 작동방식 때문에 발생한다. 브라우저는 특정 요소에서 이벤트가 발생 했을 때 그 이벤트를 화면 최상위 요소가지 이벤트를 전달 하도록 되어있다. 따라서 여기서도 최상위 요소인 class가 ‘td3’인 div 태그까지 전달 되는 것이다.

이벤트 캡쳐

이벤트 캡쳐는 이벤트 버블링과는 반대로 진행되는 이벤트 전파 방식이다.

addEventListener 객체에 capture 값을 true로 설정하면 이벤트 버블링과 반대로 작동하여 이벤트를 탐색한다. 따라서 위의 코드의 td1 class를 클릭하면 아래와 같은 결과값을 얻는다.

event.stopPropagation()

event.stopPropagation() API를 사용하면 해당 이벤트가 전파 되는 것을 막는다. 따라서 이벤트 버블링의 경우 요소에 클릭 이벤트를 주었을 때 해당 요소에만 이벤트를 발생 시키고 상위 요소로의 이벤트 전달을 막는다.
따라서 위 이벤트 캡쳐링 예제의 코드에 e.stopPropagation() api를 추가하면 td3만 출력 되게 된다.

event.preventDefault()

이벤트 본래의 동작이란 예를 들어 앵커 태그의 클릭하면 페이지를 이동하기, 텍스트 상자에서의 키 누르기라면 문자를 반영하기 등 브라우저 표준으로 정해진 동작이다. 이벤트 객체의 event.preventDefault()를 사용하면 이 동작을 취소할 수 있다.

위 코드를 실행하면 alert 창만 실행되고 페이지가 이동되지 않는 것을 알 수있다.

정리

사실 이 부분은 프론트엔드의 면접은행에 많이 나오는 부분이라 언제고 정리를 해서 공부를 해야 겠다 생각하고 있었는데 캡틴판교님의 블로그를 얼씬 거리다가 때 마침 정리를 하게 되었다. 사실 이 분의 글로 대부분의 내용을 정리했다고 봐도 과언이 아닐 것이다. 정리를 하면서도 헷갈리는 부분이 없지 않아 있지만 다시 글을 보며 스터디 그룹분들과 토론 해 봐야 겠다.
