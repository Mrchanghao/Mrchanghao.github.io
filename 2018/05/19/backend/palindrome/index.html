palindrome

이번 주 매일프로그래밍에서 온 메일을 보니 반가운 문제가 하나 왔다. palindrome문제가 온것이다.
사실 지난 주 문제의 악몽에서 아직도 벗어 나지 못해 이걸 깔끔히 포기 하고 있는 상태 였는데 그래도 간만에 아는 문제들이 나와서 다행이라는 생각이…. ㅎㅎ

위키백과에 나와 있는 panlindrome의 정의는 대략 이렇다.

A palindrome is a word, number, or other sequence of characters which reads the same backward as forward, such as madam or racecar

해석 해 보면 앞으로 읽으나 뒤로 읽으나 같은 값을 출력하는 글자열 혹은 숫자라는 뜻이다.

어? 쉬운 문제네 하고 풀었더니 역시나… 내 예상은 언제나 빗나간다. 함수 인자를 String이 아니라 number 타입으로 넣어야 하는 거구나… 그럼 그렇지 그냥 이렇게 메소드만 쓰는 문제를 줄리가 없지. 순간 한달에 10달러 하는 패키지를 그냥 구매해 버릴 까 하는 욕심이 들었지만 다시 한번 고민 해 보기로 했다.

일단 좀 지저분 하게 풀었다. ToString()도 부끄럽지만 스택오버플로우 형님들의 힘….ㅠㅠ
왜 알던 메소드들이 갑자기 생각이 안 날까? 아직도 한참 공부를 해야 한다는 것만 보여주는 문제 였다. 사실 toString() method만 알면 되는 걸 이걸 생각 한 답시고 몇 십분을 고민 하고 있었으니…

정리

이 알고리즘은 사실 string 타입으로는 많이 다루어 봤지만 이렇게 number 타입으로 다루는 건 꿈에도 생각 못 한 문제였다. 그리고 스택오버플로우를 검색 해보니 string 타입으로 문제를 풀면 약간 숫자형 보다는 퍼포먼스적으로 느리다는 충격적인 코멘트를 봤다. 이건 왜 그런지 이해가 잘 안간다.
스터디에서 물어봐야 할 것이 또 하나 늘어났다.
사실 테스트 코드도 짜서 직접 돌려 봐야 하는데 리액트 공부가 더 하고 싶어서 오늘 알고리즘 테스트 코드는 유데미 강사님의 테스트 코드 보며 있다가 짜야 겠다.
