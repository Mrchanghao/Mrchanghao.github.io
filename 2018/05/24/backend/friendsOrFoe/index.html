friend or Foe문제 라는데 문제 의 기본은 이렇다 string 타입 모음으로 된 리스트 어레이가 인수로 들어가고 이 어레이의 elements들 string 이 4글자면 새로운 어레이에 추가하고 아니면 버리는 함수를 만드는 알고리즘 문제이다. 오늘의 문제는 운 좋게도 좀 쉬운 문제가 걸렸다. 며칠 codewar에 도전하면서 느낀 바로는 좀 내 수준에 안 맞는 문제들을 많이 스킵하면 포기 하지 말고 용기 얻으라고 쉬운 문제를 냉큼 던져 주는거 같다. 하지만 쫌심 상한다고 어케 눈 앞에 있는 문제를 버릴까. 부끄러우면서도 그냥 풀었다.

역시 codewar의 장단점은 이거다 내가 문제를 풀고 테스트까지 통과하면 기분이 좋다 하지만 다른 천재들의 풀이법을 보면 난 왜 이 생각을 못 했을까 하는 생각이 든다는 것이다.
아 react 공부하면서 filter를 그렇게 많이 사용했는데 생각도 못 했다. 일단 jest로 테스트 코드 한번 작성해보고 이 두문제의 퍼포먼스를 한번 비교해 보고 싶다.
